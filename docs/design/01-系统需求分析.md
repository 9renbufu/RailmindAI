# RailMind AI - 系统需求分析

> 版本：v1.0 | 日期：2026-06-27

---

## 一、业务背景

对标 12306 铁路售票系统，构建一个具备 **AI 智能能力** 的企业级铁路售票平台。系统需要处理高并发抢票、实时余票查询、智能推荐、异常检测等核心场景，同时融入 Spring AI 提供的 LLM 能力实现智能化运营。

---

|------|---------|--------|
| 用户管理 | 注册/登录、实名认证、乘车人管理、JWT鉴权 | P0 |

| 限流熔断 | 高并发防护、服务降级、熔断器 | P0 |

## 五、业务流程概览

### 5.1 购票主流程

```
查询余票 → 选择车次 → 选择乘车人/座位 → 锁定座位 → 创建订单 → 支付 → 出票成功
```

### 5.2 退票流程

```
选择订单 → 申请退票 → 计算退票费 → 确认退款 → 释放库存 → 退款到账
```

### 5.3 改签流程

```
选择订单 → 选择新车次 → 比对差价 → 锁定新座位 → 释放旧座位 → 支付差价(或退款)
```

### 5.4 候补流程

```
余票不足 → 提交候补 → 进入队列等待 → 有退票时自动兑现 → 通知用户支付
```
