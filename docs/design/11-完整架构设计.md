## 三、请求流程

### 3.1 通用请求链路

```mermaid
sequenceDiagram
    participant C as 浏览器(Vue3)
    participant N as Nginx
    participant G as Gateway
    participant S as 业务服务
    participant R as Redis
    participant M as MySQL

    C->>N: HTTPS 请求
    N->>N: SSL终结 / 负载均衡 / IP限流
    N->>G: 转发(HTTP)

    G->>G: 1.解析路由(Path→Service)
    G->>G: 2.JWT校验(读Redis验证Token)
    G->>G: 3.全局限流(令牌桶)
    G->>G: 4.请求日志(TraceId)

    G->>S: 转发(添加X-User-Id Header)

    S->>S: 1.参数校验(@Valid)
    S->>R: 2.查缓存
    alt 缓存命中
        R-->>S: 返回缓存数据
    else 缓存未命中
        S->>M: 3.查数据库
        M-->>S: 返回数据
        S->>R: 4.写入缓存(TTL)
    end
    S->>S: 5.业务逻辑处理
    S-->>G: 返回Result<T>
    G-->>N: 返回响应
    N-->>C: 返回JSON
```

### 3.2 认证流程

```mermaid
sequenceDiagram
    participant C as 浏览器
    participant G as Gateway
    participant U as User Service
    participant R as Redis

    Note over C,R: 登录流程
    C->>G: POST /api/auth/login {phone, password}
    G->>U: 转发
    U->>U: 校验密码(BCrypt)
    U->>U: 生成AccessToken(15min) + RefreshToken(7d)
    U->>R: SET user:token:{AT} = userId EX 900
    U->>R: SET user:refresh:{RT} = userId EX 604800
    U-->>C: {accessToken, refreshToken}

    Note over C,R: 后续请求
    C->>G: GET /api/order/list (Header: Authorization: Bearer {AT})
    G->>R: GET user:token:{AT}

    G->>G: 注入X-User-Id Header
    G->>U: 转发(带X-User-Id)

    Note over C,R: Token刷新
    C->>G: POST /api/auth/refresh {refreshToken}
    G->>U: 转发
    U->>R: GET user:refresh:{RT} → 验证有效
    U->>U: 生成新AccessToken
    U->>R: 删除旧Token / 存新Token
    U-->>C: {newAccessToken}
```

### 4.2 抢票详细时序

```mermaid
sequenceDiagram
    participant U as 用户
    participant G as Gateway
    participant T as Ticket Service
    participant R as Redis
    participant K as Kafka
    participant O as Order Service
    participant P as Payment Service
    participant DB as MySQL

    U->>G: POST /api/order/create {trainNo, date, passengers}
    G->>G: 用户级限流(1次/秒)
    G->>T: 转发

    Note over T,R: 第一阶段: Redis原子扣减(毫秒级)
    T->>R: EVAL stock_deduct.lua<br/>KEYS=[stock, user_order]<br/>ARGV=[trainId, date, from, to, seatType, userId]

    alt 库存不足
        R-->>T: return 0 (库存不足)
        T-->>U: {code: 4001, msg: "余票不足"}
    else 用户重复下单
        R-->>T: return -1 (已下单)
        T-->>U: {code: 4002, msg: "请勿重复下单"}
    else 扣减成功
        R-->>T: return 1 (成功) + 可用座位号
        T-->>K: 发送 order-created 事件
        T-->>U: {code: 200, msg: "排队中", data: {orderNo}}
    end

    Note over O,DB: 第二阶段: 异步创建订单(秒级)
    K->>O: 消费 order-created 事件
    O->>DB: INSERT t_order (status=CREATED)
    O->>DB: INSERT t_order_item × N
    O->>DB: INSERT t_payment (status=PENDING)
    O->>R: SET order:timeout:{orderNo} EX 900
    O->>K: 发送 order-locked 事件

    Note over U,P: 第三阶段: 用户支付
    U->>G: POST /api/payment/pay {orderNo, payType}
    G->>P: 转发
    P->>P: 调用支付网关(模拟)
    P->>DB: UPDATE t_payment SET status=SUCCESS
    P->>K: 发送 order-paid 事件

    Note over O,DB: 第四阶段: 确认出票
    K->>O: 消费 order-paid 事件
    O->>DB: UPDATE t_order SET status=PAID
    O->>DB: UPDATE t_order_item SET seat_no=分配座位
    O->>DB: UPDATE t_order SET status=TICKETED
    O->>K: 发送 ticket-issued 事件
    K->>U: WebSocket推送: 出票成功
```

### 4.3 Redis Lua 扣库存脚本

```lua
-- stock_deduct.lua
-- KEYS[1] = ticket:stock:{trainId}:{date}:{from}:{to}:{seatType}
-- KEYS[2] = user:ordered:{trainId}:{date}:{userId}
-- ARGV[1] = userId
-- ARGV[2] = expireSeconds (900)

local stock = tonumber(redis.call('GET', KEYS[1]))
if stock == nil then
    return -2  -- 库存Key不存在，系统异常
end

if stock <= 0 then
    return 0   -- 库存不足
end

local ordered = redis.call('GET', KEYS[2])
if ordered then
    return -1  -- 用户已下单(防重复)
end

redis.call('DECR', KEYS[1])
redis.call('SETEX', KEYS[2], ARGV[2], ARGV[1])
return 1  -- 扣减成功
```

```mermaid
sequenceDiagram

    end
```

### 7.3 WebSocket 连接管理

```mermaid
sequenceDiagram
    participant U as 浏览器
    participant GW as Gateway
    participant Msg as Message Service
    participant R as Redis

    Note over U,R: 建立连接
    U->>GW: WS /ws?token={accessToken}
    GW->>R: 验证Token → userId
    GW->>Msg: 转发WebSocket连接
    Msg->>R: SET ws:user:{userId} = {serverId, sessionId}
    Msg->>U: 连接成功

    Note over U,R: 心跳保活
    loop 每30秒
        U->>Msg: PING
        Msg->>U: PONG
    end

    Note over U,R: 订阅主题
    U->>Msg: SUBSCRIBE {topic: "order:12345"}
    Msg->>R: SADD ws:subscribe:order:12345 userId
    Msg->>U: 订阅成功

    Note over U,R: 推送消息
    Msg->>R: SMEMBERS ws:subscribe:order:12345
    Msg->>U: {type:"ORDER_STATUS", data:{status:"TICKETED", seatNo:"05车12A"}}

    Note over U,R: 断线重连
    U->>U: 检测到断线
    U->>GW: WS /ws?token={token}&lastId={lastMsgId}
    GW->>Msg: 重连
    Msg->>Msg: 推送断线期间的消息(lastId之后)
```
