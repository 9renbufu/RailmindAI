## 二、AI 功能模块

### 2.1 智能客服（RAG）

```
技术栈：Spring AI + RAG + 向量数据库

架构：
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  用户    │───>│ Chat     │───>│ RAG      │───>│ LLM      │
│  问题    │    │ Controller│    │ Pipeline │    │ (GPT-4o) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              ┌──────────┐  ┌──────────┐  ┌──────────┐
              │ FAQ向量库 │  │ 政策文档  │  │ 历史对话 │
              │ (Embedding)│ │ (Embedding)│ │ 上下文  │
              └──────────┘  └──────────┘  └──────────┘

RAG流程：
1. 用户提问 → Embedding → 向量检索 Top-K
2. 拼接 Context + 用户问题 → LLM
3. LLM 生成回答（引用来源）
4. 多轮对话：维护 ConversationHistory（Redis）

知识库内容：
- 火车票购买规则
- 退票/改签政策
- 常见问题FAQ
- 站点信息
- 特殊人群购票规则
```

### 2.2 智能路线推荐

```
场景：用户输入出发地+目的地+日期，AI推荐最佳路线

Function Calling 工具：
- searchTrains(from, to, date) → 查询可用列车
- getTicketPrice(trainNo, seatType) → 查询票价
- getUserPreference(userId) → 获取用户偏好
- getTransferSuggestion(from, to, date) → 中转方案

推荐输出：
- 直达方案（按性价比/时间排序）
- 中转方案（最优中转站）
- 备选日期建议
- 票价走势预测
```

### 2.3 客流预测

```
数据源：历史售票数据、节假日日历、天气数据

方案：
1. 特征工程：提取日期特征、历史同期数据、节假日标记
2. 模型：使用 Spring AI 调用 LLM 分析趋势 + 简单时间序列
3. 输出：未来7天热门线路客流预测

应用场景：
- 运营人员：提前调配运力
- 用户端：显示"该车次预计XX%上座率"
- AI客服："建议您错峰出行，X日票源更充足"
```

### 2.4 异常订单检测

```
规则引擎 + AI 双重检测：

规则引擎（Drools / 自研）：
- 同一用户短时间大量下单
- 同一身份证频繁购票退票
- 异常IP段集中下单

AI检测（Spring AI + LLM）：
- 将订单特征序列化为 Prompt
- LLM 判断是否为黄牛/刷票行为
- 返回风险等级 + 理由

输出：风控决策（放行 / 人工审核 / 拦截）
```

## 三、Spring AI 配置

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      base-url: ${OPENAI_BASE_URL}  # 支持自定义端点
      chat:
        options:
          model: gpt-4o
          temperature: 0.7
          max-tokens: 2000
    vectorstore:
      type: simple  # 开发阶段用内存，生产换 Milvus/PGVector
    chat:
      memory:
        repository:
          type: redis  # 对话历史存Redis
```
