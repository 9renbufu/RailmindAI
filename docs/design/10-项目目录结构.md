## DDD 分层说明

每个服务模块内部遵循 DDD 四层架构：

```
controller/     ← 展示层：接收请求，参数校验，返回结果
    │
service/        ← 应用层：编排领域服务，事务管理
    │
domain/         ← 领域层：核心业务逻辑，实体，值对象，领域服务
    │
repository/     ← 基础设施层：数据访问，外部服务调用
```
