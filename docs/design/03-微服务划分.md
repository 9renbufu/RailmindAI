# RailMind AI - 微服务划分

> 版本：v1.0 | 日期：2026-06-27

---

## 一、架构决策

采用 **模块化单体优先，渐进拆分微服务** 的策略：

- 开发阶段：多模块 Maven 项目，模块间通过接口调用
- 部署阶段：可整体部署为单体，也可按模块拆分为独立服务
- 核心高频模块（票务、订单）预留异步消息解耦

## 三、服务职责

| 服务名 | 端口 | 职责 | 核心依赖 |
|--------|------|------|----------|
| `railmind-gateway` | 8080 | API网关、路由、限流、JWT校验 | Redis |
| `railmind-user` | 8081 | 用户注册/登录、实名认证、乘车人 | MySQL, Redis |
| `railmind-train` | 8082 | 车次/站点/时刻表/票价管理 | MySQL, Redis |
| `railmind-ticket` | 8083 | 余票查询、库存扣减、座位锁定、候补 | MySQL, Redis, Kafka |
| `railmind-order` | 8084 | 订单CRUD、退票、改签、状态机 | MySQL, Redis, Kafka |
| `railmind-payment` | 8085 | 支付、退款、对账 | MySQL, Kafka |
| `railmind-message` | 8086 | WebSocket、消息推送、公告 | Redis, Kafka |
| `railmind-ai` | 8087 | 智能推荐、客服、预测、异常检测 | Spring AI, MySQL |
| `railmind-admin` | 8088 | 后台管理、运营统计、大屏 | MySQL |

---

## 四、服务间通信

| 调用方 | 被调用方 | 通信方式 | 场景 |
|--------|---------|---------|------|
| gateway | 所有服务 | HTTP (同步) | API路由 |
| order | ticket | HTTP (同步) | 锁定库存 |
| order | payment | Kafka (异步) | 发起支付 |
| payment | order | Kafka (异步) | 支付结果回调 |
| ticket | message | Kafka (异步) | 余票变更通知 |
| order | message | Kafka (异步) | 订单状态推送 |
| ai | train | HTTP (同步/Feign) | 查询车次信息 |
| ai | user | HTTP (同步/Feign) | 查询用户画像 |

## 五、Maven 模块结构

```
railmind/
├── pom.xml                          (父POM, 统一依赖管理)
├── railmind-common/                 (公共模块: 工具类/异常/常量/配置)
├── railmind-gateway/                (网关)
├── railmind-user/                   (用户)
├── railmind-train/                  (车次)
├── railmind-ticket/                 (票务)
├── railmind-order/                  (订单)
├── railmind-payment/                (支付)
├── railmind-message/                (消息)
├── railmind-ai/                     (AI)
└── railmind-admin/                  (管理后台)
```

### 模块依赖关系

```
common <── gateway
common <── user
common <── train
common <── ticket   (依赖 train API)
common <── order    (依赖 ticket API, payment API)
common <── payment
common <── message
common <── ai       (依赖 train API, user API)
common <── admin
```
