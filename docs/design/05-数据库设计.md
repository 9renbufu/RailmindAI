## 三、完整建表 DDL

### 3.1 用户表 `t_user`

```sql
CREATE TABLE t_user (
    id              BIGINT       NOT NULL AUTO_INCREMENT COMMENT '用户ID(雪花)',
    username        VARCHAR(50)  NOT NULL COMMENT '用户名',
    password        VARCHAR(255) NOT NULL COMMENT '密码(BCrypt)',
    real_name       VARCHAR(50)  DEFAULT NULL COMMENT '真实姓名',
    id_card         VARCHAR(64)  DEFAULT NULL COMMENT '身份证号(AES加密)',
    phone           VARCHAR(20)  NOT NULL COMMENT '手机号',
    email           VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    avatar          VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
    user_level      TINYINT      NOT NULL DEFAULT 1 COMMENT '用户等级: 1-普通 2-银卡 3-金卡 4-钻石',
    status          TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 0-禁用 1-正常 2-冻结',
    last_login_time DATETIME     DEFAULT NULL COMMENT '最后登录时间',
    last_login_ip   VARCHAR(45)  DEFAULT NULL COMMENT '最后登录IP(支持IPv6)',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted         TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除: 0-正常 1-已删除',
    PRIMARY KEY (id),
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_phone (phone),
    INDEX idx_email (email),
    INDEX idx_id_card (id_card),
    INDEX idx_status (status, deleted)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**字段设计说明：**

| 字段 | 类型选择理由 |
|------|-------------|
| `id` | BIGINT 而非 INT：百万级用户 + 雪花ID 需要 64 位 |
| `password` | VARCHAR(255)：BCrypt 哈希固定 60 字符，预留空间 |
| `id_card` | VARCHAR(64)：AES-256 加密后为 Base64，约 44 字符 |
| `phone` | UNIQUE：一个手机号只能注册一个账号 |
| `user_level` | 用于候补优先级计算，避免额外关联表 |
| `last_login_ip` | VARCHAR(45)：兼容 IPv6 最长 45 字符 |

### 3.2 乘车人表 `t_passenger`

```sql
CREATE TABLE t_passenger (
    id          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '乘车人ID',
    user_id     BIGINT       NOT NULL COMMENT '所属用户ID',
    name        VARCHAR(50)  NOT NULL COMMENT '姓名',
    id_card     VARCHAR(64)  NOT NULL COMMENT '身份证号(AES加密)',
    id_card_hash VARCHAR(64) NOT NULL COMMENT '身份证号SHA256哈希(用于唯一性校验)',
    phone       VARCHAR(20)  DEFAULT NULL COMMENT '手机号',
    type        TINYINT      NOT NULL DEFAULT 1 COMMENT '类型: 1-成人 2-儿童 3-学生 4-军人 5-残疾军人',
    status      TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 0-禁用 1-正常',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted     TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除',
    PRIMARY KEY (id),
    INDEX idx_user_id (user_id, deleted),
    INDEX idx_id_card_hash (id_card_hash)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='乘车人表';
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `id_card` | AES 加密存储，不可直接查询 |
| `id_card_hash` | SHA256 哈希，用于**唯一性校验**（同一身份证不能重复添加）和精确查找 |
| `type` | 儿童票半价、学生票75折、军人票半价，票价引擎读取此字段计算 |

### 3.3 站点表 `t_station`

```sql
CREATE TABLE t_station (
    id          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '站点ID',
    code        VARCHAR(10)  NOT NULL COMMENT '站点电报码(BJP=北京)',
    name        VARCHAR(50)  NOT NULL COMMENT '站名',
    short_name  VARCHAR(20)  NOT NULL COMMENT '站名简称(北京)',
    city        VARCHAR(50)  NOT NULL COMMENT '所属城市',
    province    VARCHAR(50)  NOT NULL COMMENT '省份',
    longitude   DECIMAL(10,6) DEFAULT NULL COMMENT '经度(WGS84)',
    latitude    DECIMAL(10,6) DEFAULT NULL COMMENT '纬度(WGS84)',
    bureau      VARCHAR(50)  DEFAULT NULL COMMENT '所属铁路局',
    grade       TINYINT      NOT NULL DEFAULT 3 COMMENT '等级: 1-特等 2-一等 3-二等 4-三等 5-四等',
    status      TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-正常',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_code (code),
    INDEX idx_city (city),
    INDEX idx_province (province),
    INDEX idx_name (name),
    FULLTEXT INDEX ft_name (name, short_name)  -- 全文索引，支持模糊搜索站名
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='站点表';
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `code` | 电报码，全国唯一，3-5位大写字母，如 BJP(北京)、SHH(上海) |
| `short_name` | 前端搜索用，输入"北京"匹配比"北京站"更高效 |
| `grade` | 特等站(北京/上海/广州)、一等站(省会)等，影响排序展示 |
| `longitude/latitude` | 支持"附近车站"功能和地图展示 |

### 3.4 列车模板表 `t_train`

> **注意**：此表存储列车的"车型"信息（固定属性），不涉及具体日期。

```sql
CREATE TABLE t_train (
    id              BIGINT       NOT NULL AUTO_INCREMENT COMMENT '列车ID',
    train_no        VARCHAR(20)  NOT NULL COMMENT '车次号(G1234/D2345/K1234)',
    train_type      CHAR(1)      NOT NULL COMMENT '类型: G-高铁 D-动车 C-城际 Z-直达 T-特快 K-快速 L-临客',
    start_station   VARCHAR(10)  NOT NULL COMMENT '始发站编码',
    end_station     VARCHAR(10)  NOT NULL COMMENT '终到站编码',
    departure_time  TIME         NOT NULL COMMENT '始发站发车时间',
    arrival_time    TIME         NOT NULL COMMENT '终到站到达时间',
    run_days        TINYINT      NOT NULL DEFAULT 1 COMMENT '运行天数(跨天=2)',
    total_mileage   INT          NOT NULL COMMENT '总里程(km)',
    total_seats     INT          NOT NULL DEFAULT 0 COMMENT '总座位数',
    sale_start_days TINYINT      NOT NULL DEFAULT 15 COMMENT '预售天数(15天)',
    status          TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 0-停运 1-正常',
    remark          VARCHAR(500) DEFAULT NULL COMMENT '备注',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted         TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除',
    PRIMARY KEY (id),
    UNIQUE KEY uk_train_no (train_no),
    INDEX idx_type (train_type),
    INDEX idx_start_end (start_station, end_station),
    INDEX idx_status (status, deleted)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='列车模板表(车型信息)';
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `train_no` | 车次号，如 G1、D313、K456，同一车次号在全球唯一 |
| `train_type` | 影响票价系数、座位类型、运行速度 |
| `sale_start_days` | 预售天数，不同车次可配置不同（普通15天，临客5天） |
| `total_seats` | 冗余字段，快速判断该车次总运力 |

### 3.5 列车站序表 `t_train_station`

> 描述某列车模板经停哪些站、顺序、时刻。

```sql
CREATE TABLE t_train_station (
    id              BIGINT       NOT NULL AUTO_INCREMENT COMMENT 'ID',
    train_id        BIGINT       NOT NULL COMMENT '列车ID',
    station_code    VARCHAR(10)  NOT NULL COMMENT '站点编码',
    station_order   INT          NOT NULL COMMENT '站序(1=始发站)',
    arrival_time    TIME         DEFAULT NULL COMMENT '到达时间(始发站NULL)',
    departure_time  TIME         DEFAULT NULL COMMENT '出发时间(终到站NULL)',
    stop_duration   SMALLINT     NOT NULL DEFAULT 0 COMMENT '停靠时长(分钟)',
    mileage         INT          NOT NULL COMMENT '累计里程(km)',
    is_start        TINYINT      NOT NULL DEFAULT 0 COMMENT '是否始发站',
    is_end          TINYINT      NOT NULL DEFAULT 0 COMMENT '是否终到站',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_train_order (train_id, station_order),
    INDEX idx_train_id (train_id),
    INDEX idx_station_code (station_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='列车站序表(列车-站点关系)';
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `station_order` | 从 1 开始递增，同一车次内唯一 |
| `mileage` | 从始发站开始的累计里程，用于计算票价 |
| `is_start/is_end` | 冗余标记，避免查询时额外判断 order=1 或 order=max |

### 3.6 座位类型表 `t_seat_type`

> 描述某列车有哪些座位类型及票价系数。

```sql
CREATE TABLE t_seat_type (
    id              BIGINT        NOT NULL AUTO_INCREMENT COMMENT 'ID',
    train_id        BIGINT        NOT NULL COMMENT '列车ID',
    seat_type_code  VARCHAR(10)   NOT NULL COMMENT '座位编码(SW/YW/YZ等)',
    seat_type_name  VARCHAR(20)   NOT NULL COMMENT '座位名称',
    seat_count      INT           NOT NULL COMMENT '座位数',
    price_per_km    DECIMAL(6,2)  NOT NULL COMMENT '每公里单价(元)',
    created_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_train_seat (train_id, seat_type_code),
    INDEX idx_train_id (train_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='座位类型表';
```

**座位编码标准（国标）：**

| 编码 | 名称 | 说明 |
|------|------|------|
| SW | 商务座 | 高铁特有，票价系数最高 |
| YW | 一等座 | 高铁/动车 |
| ZE | 二等座 | 高铁/动车，最常见 |
| YZ | 硬座 | 普速列车 |
| RW | 软卧 | 普速列车 |
| YW | 硬卧 | 普速列车 |
| RZ | 软座 | 普速列车 |

**票价计算公式：** `票价 = price_per_km × 区间里程 × 折扣系数`

### 3.7 车次运行计划表 `t_train_schedule`

> **核心表**：描述"哪列火车在哪天运行"，是库存和订单的时间维度基础。

```sql
CREATE TABLE t_train_schedule (
    id              BIGINT       NOT NULL AUTO_INCREMENT COMMENT '运行计划ID',
    train_id        BIGINT       NOT NULL COMMENT '列车ID',
    train_no        VARCHAR(20)  NOT NULL COMMENT '车次号(冗余)',
    travel_date     DATE         NOT NULL COMMENT '运行日期',
    start_station   VARCHAR(10)  NOT NULL COMMENT '始发站编码',
    end_station     VARCHAR(10)  NOT NULL COMMENT '终到站编码',
    departure_time  DATETIME     NOT NULL COMMENT '始发站发车时间(日期+时间)',
    arrival_time    DATETIME     NOT NULL COMMENT '终到站到达时间(日期+时间)',
    status          TINYINT      NOT NULL DEFAULT 1 COMMENT '状态: 0-停运 1-正常 2-晚点 3-取消',
    delay_minutes   INT          NOT NULL DEFAULT 0 COMMENT '晚点时长(分钟)',
    remark          VARCHAR(500) DEFAULT NULL COMMENT '备注(如:临时停运原因)',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_train_date (train_id, travel_date),
    INDEX idx_travel_date (travel_date),
    INDEX idx_train_no_date (train_no, travel_date),
    INDEX idx_departure (departure_time),
    INDEX idx_status_date (status, travel_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='车次运行计划表';
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `train_id` + `travel_date` | 联合唯一：同一列车每天只有一条运行计划 |
| `departure_time` / `arrival_time` | DATETIME 而非 TIME：因为跨天运行时到达时间是次日 |
| `status` | 运营人员可标记停运/晚点，触发通知 |
| `train_no` | 冗余字段：避免查询时 JOIN t_train |

**数据量预估：** 5000列火车 × 365天 = **182.5万/年**，可按月分表。

### 3.8 票务库存表 `t_ticket_inventory`

> **核心高频表**：每个区间每种座位类型的剩余票数。

```sql
CREATE TABLE t_ticket_inventory (
    id              BIGINT       NOT NULL AUTO_INCREMENT COMMENT 'ID',
    schedule_id     BIGINT       NOT NULL COMMENT '运行计划ID',
    train_no        VARCHAR(20)  NOT NULL COMMENT '车次号(冗余)',
    travel_date     DATE         NOT NULL COMMENT '乘车日期',
    from_station    VARCHAR(10)  NOT NULL COMMENT '出发站编码',
    to_station      VARCHAR(10)  NOT NULL COMMENT '到达站编码',
    seat_type_code  VARCHAR(10)  NOT NULL COMMENT '座位类型编码',
    total_count     INT          NOT NULL COMMENT '总票数(该区间)',
    sold_count      INT          NOT NULL DEFAULT 0 COMMENT '已售数量',
    locked_count    INT          NOT NULL DEFAULT 0 COMMENT '锁定数量(预占)',
    version         BIGINT       NOT NULL DEFAULT 0 COMMENT '乐观锁版本号',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
```

**字段设计说明：**

| 字段 | 说明 |
|------|------|
| `schedule_id` | 关联运行计划，而非 train_id，保证日期维度准确 |
| `from_station` / `to_station` | 区间：A→B 与 A→C 是不同库存记录 |
| `total_count` | 该区间该座位类型的**初始可售数量**（分段限售计算后） |
| `sold_count` | 已确认售出的数量 |
| `locked_count` | 被用户锁定（预占）但尚未支付的数量 |
| `version` | 乐观锁：`UPDATE ... SET sold_count=sold_count+1 WHERE version=#{v}` |

**库存可用公式：** `可用 = total_count - sold_count - locked_count`

**分段限售说明：**
- 车次 A→B→C，总座位 100
- A→C 分配 80 张，A→B 分配 30 张，B→C 分配 30 张
- A→B 售罄后，A→C 仍有票可卖（动态调整）

## 四、索引设计总览

### 4.1 全局索引设计原则

| 原则 | 说明 |
|------|------|
| 最左前缀 | 联合索引按查询频率从左到右排列 |
| 覆盖索引 | 高频查询尽量让索引包含 SELECT 的字段 |
| 避免冗余 | `(a,b)` 已覆盖 `(a)`，无需重复创建 |
| 控制数量 | 单表索引不超过 6 个（写入性能考虑） |
| 区分度优先 | 区分度高的列放联合索引左侧 |

### 4.2 各表索引分析

#### t_order（最核心）


### 5.2 核心查询 SQL

#### 余票查询（最高频）

```sql
-- 查询某车次某日从A站到B站的各座位类型余票
SELECT
    ti.seat_type_code,
    st.seat_type_name,
    ti.total_count - ti.sold_count - ti.locked_count AS available_count
FROM t_ticket_inventory ti
JOIN t_seat_type st ON st.train_id = (
    SELECT train_id FROM t_train_schedule WHERE id = ti.schedule_id
) AND st.seat_type_code = ti.seat_type_code
WHERE ti.train_no = 'G1234'
  AND ti.travel_date = '2026-07-15'
  AND ti.from_station = 'BJP'
  AND ti.to_station = 'SHH';
```

**优化方案：**
```sql
-- 优化1: 冗余 seat_type_name 到 t_ticket_inventory，避免 JOIN
-- 优化2: 使用 Redis 缓存，TTL 30s
-- 优化3: 本地缓存 Caffeine，TTL 5s

-- 最终SQL（无需JOIN）
SELECT seat_type_code, seat_type_name,
       (total_count - sold_count - locked_count) AS available_count
FROM t_ticket_inventory
WHERE train_no = 'G1234'
  AND travel_date = '2026-07-15'
  AND from_station = 'BJP'
  AND to_station = 'SHH';
```

#### 用户订单列表

```sql
-- 查询用户最近的订单
SELECT order_no, train_no, travel_date, from_station_name, to_station_name,
       pay_amount, status, created_at
FROM t_order
WHERE user_id = 123456
  AND deleted = 0
ORDER BY created_at DESC
LIMIT 20;
```

**优化方案：**
```sql
-- idx_user_id_created (user_id, created_at DESC) 覆盖排序
-- LIMIT 20 深度分页优化: 使用游标分页
WHERE user_id = 123456 AND created_at < '上次最后一条的时间'
ORDER BY created_at DESC LIMIT 20
```

#### 库存扣减（下单核心）

```sql
-- 乐观锁扣减
UPDATE t_ticket_inventory
SET sold_count = sold_count + 1,
    version = version + 1,
    updated_at = NOW()
WHERE id = 1001
  AND (total_count - sold_count - locked_count) > 0
  AND version = 5;
-- 影响行数=0 表示库存不足或并发冲突
```

#### 过期锁座扫描

```sql
-- 定时任务: 每分钟扫描过期的锁座记录
SELECT id, schedule_id, seat_type_code, from_station, to_station
FROM t_seat_lock
WHERE expire_time < NOW()
  AND status = 1
LIMIT 1000;

-- 批量释放
UPDATE t_seat_lock
SET status = 0,
    release_reason = 1,
    updated_at = NOW()
WHERE expire_time < NOW()
  AND status = 1
LIMIT 1000;
```

```sql
FROM t_waitlist
WHERE schedule_id = 1001
  AND seat_type_code = 'ZE'
  AND status = 'WAITING'

LIMIT 10;
```

## 七、SQL 优化建议

### 7.1 通用优化

| 优化项 | 说明 |
|--------|------|
| **避免 SELECT \*** | 只查需要的字段，减少网络传输和内存消耗 |
| **避免大事务** | 下单流程拆分为 Redis 原子操作 + Kafka 异步落库 |
| **批量操作** | 库存初始化、数据迁移使用批量 INSERT |
| **LIMIT 必加** | 任何列表查询必须有 LIMIT，防止全表扫描 |
| **软删除索引** | `deleted` 字段必须包含在查询条件的索引中 |

### 7.2 高频查询优化

```sql
-- ❌ 错误: 深度分页
SELECT * FROM t_order WHERE user_id = ? ORDER BY created_at DESC LIMIT 10000, 20;

-- ✅ 正确: 游标分页
SELECT * FROM t_order
WHERE user_id = ? AND created_at < '上一页最后一条的时间'
ORDER BY created_at DESC
LIMIT 20;
```

```sql
-- ❌ 错误: 非索引列排序
SELECT * FROM t_order WHERE user_id = ? ORDER BY total_amount DESC;

-- ✅ 正确: 如果需要按金额排序，添加索引
ALTER TABLE t_order ADD INDEX idx_user_amount (user_id, total_amount DESC);
```

```sql
-- ❌ 错误: OR 条件导致索引失效
SELECT * FROM t_station WHERE name LIKE '%北京%' OR city LIKE '%北京%';

-- ✅ 正确: 使用全文索引
SELECT * FROM t_station WHERE MATCH(name, short_name) AGAINST('北京' IN BOOLEAN MODE);
```

### 7.3 写入优化

```sql
-- ❌ 错误: 逐行更新库存
UPDATE t_ticket_inventory SET sold_count = sold_count + 1 WHERE id = 1;
UPDATE t_ticket_inventory SET sold_count = sold_count + 1 WHERE id = 2;

-- ✅ 正确: 批量更新
UPDATE t_ticket_inventory
SET sold_count = sold_count + 1, version = version + 1
WHERE id IN (1, 2) AND version = #{expectedVersion};
```

### 7.4 表设计优化

| 优化项 | 说明 |
|--------|------|
| **合理使用冗余** | `from_station_name`、`train_no` 等高频查询字段冗余存储，避免 JOIN |
| **JSON 字段** | 扩展信息使用 JSON 而非 EAV 模式（如 `extra`、`metadata`） |
| **DECIMAL 而非 FLOAT** | 金额字段必须用 DECIMAL，避免浮点精度问题 |
| **VARCHAR 长度** | 按实际需求设定，不要一律 VARCHAR(255) |
| **NOT NULL** | 尽量 NOT NULL + DEFAULT，减少 NULL 判断开销 |
| **字符集统一** | 全部使用 utf8mb4_unicode_ci，避免 JOIN 时字符集转换 |
