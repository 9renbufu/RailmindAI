# RailMind AI - 功能模块划分

> 版本：v1.0 | 日期：2026-06-27

---

## 一、模块总览

```
RailMind AI
├── 用户域 (User Domain)
├── 车次域 (Train Domain)
├── 票务域 (Ticket Domain)
├── 订单域 (Order Domain)
├── 支付域 (Payment Domain)
├── 消息域 (Message Domain)
├── AI 域 (AI Domain)
└── 运营域 (Operation Domain)
```

---

## 二、用户域 (User Domain)

| 功能 | 说明 |
|------|------|
| 手机号/邮箱注册 | 验证码校验，密码加密存储（BCrypt） |
| JWT 登录 | Access Token（15min）+ Refresh Token（7d），Redis 存储会话 |
| 实名认证 | 身份证号 + 姓名校验（模拟公安部接口） |
| 乘车人管理 | 常用联系人CRUD，最多15人 |
| 用户画像 | 基于购票历史构建偏好标签，供 AI 推荐使用 |

---

## 三、车次域 (Train Domain)

| 功能 | 说明 |
|------|------|
| 车次管理 | G/D/Z/K/T 等类型车次 CRUD |
| 站点管理 | 全国火车站数据，经纬度、所属路局 |
| 途经站 | 车次经过的站点及时刻，支持跨天运行 |
| 票价引擎 | 按里程 × 座位类型 × 车次类型计算，支持浮动定价 |
| 运行图 | 按日期排列车次，支持加开/停运 |

---

## 四、票务域 (Ticket Domain)

| 功能 | 说明 |
|------|------|
| 余票查询 | 出发站→到达站→日期，支持中转联程 |
| 库存模型 | 分段限售：A→C 的票可拆为 A→B + B→C |
| 座位锁定 | Redis 分布式锁 + 数据库乐观锁双重保障 |
| 候补队列 | Redis Sorted Set 按优先级排序，定时扫描兑现 |

---

## 五、订单域 (Order Domain)

| 功能 | 说明 |
|------|------|
| 下单流程 | 选座 → 锁票 → 创建订单 → 支付 → 出票 |
| 超时取消 | 下单后15分钟未支付自动取消，释放库存 |
| 退票 | 按退票时间阶梯扣费 |
| 改签 | 同车次改签 + 跨车次改签，需重新选座 |

### 订单状态机

```
                    ┌──────────────────────────────────────────────┐
                    │                                              │
                    ▼                                              │
 ┌─────────┐  下单  ┌─────────┐  支付   ┌─────────┐  出票  ┌───────────┐
 │ CREATED │──────>│ PAYING  │───────>│  PAID   │──────>│ TICKETED  │
 │ 待支付   │       │ 支付中   │        │ 已支付   │       │  已出票    │
 └─────────┘       └─────────┘        └─────────┘       └───────────┘
      │                 │                  │                    │
      │ 超时/取消        │ 支付失败          │ 退票               │ 完成
      ▼                 ▼                  ▼                    ▼
 ┌──────────┐    ┌──────────┐      ┌───────────┐       ┌───────────┐
 │CANCELLED │    │ PAY_FAIL │      │REFUNDING  │       │ COMPLETED │
 │  已取消   │    │ 支付失败  │      │  退款中    │       │  已完成    │
 └──────────┘    └──────────┘      └───────────┘       └───────────┘
                                      │    │
                                  退款成功  退款失败
                                      ▼    ▼
                               ┌────────┐ ┌────────┐
                               │REFUNDED│ │PAY_FAIL│
                               │ 已退款  │ │        │
                               └────────┘ └────────┘
```

---

## 六、支付域 (Payment Domain)

| 功能 | 说明 |
|------|------|
| 支付单管理 | 创建支付单、状态流转 |
| 模拟支付网关 | 模拟微信/支付宝支付（开发阶段） |
| 退款处理 | 退票触发退款，原路退回 |
| 对账 | 定时对账，确保订单与支付金额一致 |

---

## 七、消息域 (Message Domain)

| 功能 | 说明 |
|------|------|
| WebSocket 连接管理 | 长连接管理、心跳检测、断线重连 |
| 余票变更通知 | 退票释放时通知候补用户 |
| 订单状态推送 | 支付成功、出票成功等实时推送 |
| 系统公告 | 运营发布公告，全站/定向推送 |
| Kafka 消息总线 | 异步事件驱动，服务间解耦 |

---

## 八、AI 域 (AI Domain)

| 功能 | 说明 |
|------|------|
| 智能路线推荐 | 根据用户需求推荐直达/中转方案 |
| 智能客服 | RAG 检索增强生成，多轮对话 |
| 客流预测 | 基于历史数据预测未来客流 |
| 异常订单检测 | 规则引擎 + AI 双重检测黄牛/刷票 |
| 票价动态建议 | 根据供需关系建议浮动定价 |

---

## 九、运营域 (Operation Domain)

| 功能 | 说明 |
|------|------|
| 售票统计报表 | 按车次/线路/日期统计售票数据 |
| 实时大屏 | 核心指标实时展示 |
| 日志与审计 | 操作日志、审计追踪 |
| 系统配置管理 | 系统参数、限流规则等配置 |
